Mensajes de Estado HTTP

  • 100 Continue: The server has received the request headers, and the client should proceed to send the request body
  • 101 Switching Protocols: The requester has asked the server to switch protocols
  • 103 Early Hints: Used with the Link header to allow the browser to start preloading resources while the server prepares a response
  • 200 OK: The request was successful
  • 301 Moved Permanently: The requested resource has been permanently moved to a new location
  • 404 Not Found: The requested resource could not be found
  • 500 Internal Server Error: The server encountered an unexpected condition that prevented it from fulfilling the request

Códigos de estado de respuesta

El elemento de código de estado es un código entero de tres dígitos que indica el resultado del intento de comprender y satisfacer la solicitud.

Los códigos de estado HTTP son extensibles. No es necesario que los clientes HTTP comprendan el significado de todos los códigos de estado registrados, aunque tal comprensión obviamente es deseable. Sin embargo, el cliente DEBE comprender la clase de cualquier código de estado, como lo indica el primer dígito y tratar un código de estado no reconocido como equivalente al código de estado x00 de esa clase, con la excepción de que un destinatario NO DEBE almacenar en caché una respuesta con un código de estado no reconocido.

Por ejemplo, si un código de estado no reconocido de 471 es recibido por un cliente, el cliente puede asumir que había algo mal con su solicitud y tratar la respuesta como si hubiera recibido un 400 (Mal) Código de estado de la solicitud). El mensaje de respuesta normalmente contendrá una representación que explica el estado.

El primer dígito del código de estado define la clase de respuesta. Los dos últimos dígitos no tienen ninguna función de categorización. Hay cinco valores para el primer dígito:
